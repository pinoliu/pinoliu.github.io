Linux系统状态查询命令

发表于 2018-12-04 | Edited on 2019-05-10 | Comments： | Views：

简述

当程序在Linux系统中运行时，遇到一些性能资源的瓶颈的时候，就需要通过分析Linux系统本身的数据来分析具体的优化点。如具体哪个资源（io、cpu、memory、cache等）、哪个线程（业务、平台等）对整体的负担太大，需要进行优化。

常用指令

CPU

pidstat -u 1

Memory

pidstat -r 1

IO

pidstat -d

每个CPU的使用情况

如果单个CPU的占用率特别高，那么有可能是单一线程引起的。

mpstat -P ALL 1

统计并输出CPU实用信息及特定设备或分区的IO信息

iostat

iostat -xz 1

聚焦机器磁盘 IO 情况

磁盘使用情况

df -h

文件大小

du -sh *

系统健康

top

机器负载

CPU资源的进程和阻塞在不可终端IO进程（进程状态为D），1min、5min、15min的平均负载情况

uptime

系统核心指标

vmstat 1
